# Paper Trading Admin

一块给老板看的**模拟盘实时监控墙**。它跑在一台常开的机器上,自动连上局域网里每个同事正在用的 paper trading app,把所有人的**净值、盈亏、当日表现、持仓、成交**汇总到一个页面,实时刷新,还能排名、下钻看明细、远程帮新同事开户。

它**自己不交易**,只看不动手——监控这台机器关了,大家照常交易,互不影响。

## 它能做什么

- **监控墙**:每个同事一张卡片,净值曲线 + 总收益率 + 当日盈亏 + 仓位,绿涨红跌一眼看清
- **排行榜**:按收益率自动排名
- **下钻**:点开任意一张卡,看他的账户、持仓明细、最近成交
- **掉线提醒**:谁的 app 断了 / 又上线了,顶部告警立刻知道
- **远程开户**:新同事来了,在这页上直接给他的机器建账户,不用跑过去
- **真·实时**:有成交马上更新(秒级),不是傻等刷新

## 怎么用

### 方式一:下载现成的(推荐)

到 [Releases](../../releases) 下载对应系统的压缩包,解压双击运行:

- **macOS**:`ptadmin-macos-arm64.zip` → 双击 `ptadmin`
- **Windows**:`ptadmin-windows-x64.zip` → 双击 `ptadmin.exe`

启动后浏览器会自动打开 `http://localhost:8800`。

### 方式二:用 Python 跑源码

需要 Python 3.10 以上,无任何第三方依赖:

```bash
python3 -m admin
```

然后浏览器打开 `http://localhost:8800`。

## 让同事的 app 接进来

同事那边**几乎不用改**。只要他启动 app 时让局域网能访问:

```bash
HOST=0.0.0.0 python3 -m backend.server
```

然后在监控墙点右上角 **「+ 添加节点」**,填上他的名字和地址(比如 `http://192.168.1.23:8000`)就行。

想要"自动注册、免填 IP""成交秒级推送""开户加密码保护"这些进阶功能,给同事的 app 打个小补丁即可,**默认关闭、完全向后兼容**,做法见 [`node_patch/PATCH.md`](node_patch/PATCH.md)。

## 安全 / 隔离

- 监控墙只会**读**同事的数据;唯一的"写"操作是你亲手点"开户",经过确认才发出去。
- 监控墙的机器宕机,**不影响任何人交易**。
- 某个同事的 app 掉线了,墙上把他标成离线,并显示他**最后一次的已知数据**,不会一片空白。

## 配置(可选,环境变量)

不设也能跑,默认值都合理。

| 变量 | 默认 | 说明 |
|---|---|---|
| `ADMIN_PORT` | `8800` | 监控墙端口 |
| `ADMIN_TOKEN` | 空 | 设了之后,"开户"等写操作需要这个口令 |
| `POLL_INTERVAL` | `3.0` | 多少秒拉一次数据(2–5 合适) |
| `ADMIN_DB` | `data/admin.db` | 监控墙自己的数据库位置 |

完整列表见 [`admin/config.py`](admin/config.py)。
